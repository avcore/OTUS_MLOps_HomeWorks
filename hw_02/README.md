## **Цель работы**
В рамках задания была спроектирована и реализована облачная инфраструктура для обработки транзакционных данных с использованием сервисов Yandex Cloud:
- Object Storage (S3)
- Yandex Data Processing (Spark)
- HDFS
- Terraform (Infrastructure as Code)

## **Архитектура решения**

```
Public S3 (otus-mlops-source-data)
                │
                ▼

                │

   Yandex Object Storage (bucket)
                │
                ▼
      Hadoop distcp (s3a)
                │
                ▼
             HDFS
                │
                ▼
        Spark Cluster
```

## **Используемые технологии**
- **Terraform** — управление инфраструктурой
- **Yandex Cloud**:
    - Object Storage
    - DataProc (Spark + HDFS)
- **s3cmd** — копирование данных
- **Hadoop distcp** — перенос данных в HDFS
- **SSH** — доступ к master-узлу

## **Выполнение - обязательных заданий**

### **1. Создание bucket (Terraform)**

Создан S3 bucket с использованием Terraform:
```
resource "yandex_storage_bucket" "mlops_bucket" {
  bucket = var.bucket_name
}
```
Bucket используется как промежуточное хранилище данных.
### **2. Копирование данных через s3cmd**

Использован скрипт:
```
./scripts/s3cmd_sync.sh <bucket_name>
```

Источник данных:
```
s3://otus-mlops-source-data/
```

Результат:
- данные успешно скопированы в созданный bucket

### **3. Создание Spark-кластера (DataProc)**

Кластер развёрнут через Terraform:

|**Подкластер**|**Конфигурация**|
|---|---|
|Master|s3-c2-m8, 40 GB|
|Data|3 × s3-c4-m16, 128 GB|

Пример ресурса:
```
resource "yandex_dataproc_cluster" "cluster" {
  ...
}
```
### **4. Перенос данных в HDFS (distcp)**

Выполнено подключение по SSH к master-узлу и запуск:
```
./scripts/hdfs_distcp.sh <bucket_name>
```
Команда внутри:
```
hadoop distcp s3a://<bucket> /data
```
После выполнения:
```
hdfs dfs -ls /data
```

### **5. Оценка стоимости (Yandex Cloud)**
В рамках выполнения задания был развернут кластер Yandex Data Proc для обработки данных с использованием Spark и HDFS.
#### **💰 Стоимость кластера**
Согласно расчётам Terraform:
- **Почасовая стоимость кластера:**
    0.354228 USD/hour
- **Месячная стоимость (24/7):**
    255.04 USD/month

Это включает:
- master node (s3-c2-m8)
- 3 data node (s3-c4-m16)
- диски (network-hdd)

#### **Стоимость хранения**
|**Тип хранения**|**Стоимость / месяц**|
|---|---|
|HDFS|12.82 USD|
|Object Storage|7.67 USD|

#### **Сравнение HDFS vs Object Storage**
Коэффициент:
`HDFS / Object Storage = 1.67`

Вывод:
- HDFS **дороже примерно на 67%**
- Причина:
    - репликация (обычно ×3)
    - привязка к compute (диски на нодах)
    - постоянная оплата VM

Object Storage:
- дешевле
- масштабируемый
- не требует compute

HDFS:
- нужен для Spark (локальность данных)
- быстрее для некоторых задач

#### **Вывод по затратам**
- Основные расходы — это **compute (кластер)**, а не storage
- Хранение в HDFS — **второстепенная статья затрат**
- Использование Object Storage значительно снижает стоимость хранения

|**Хранилище**|**Стоимость**|**Вывод**|
|---|---|---|
|HDFS|Высокая|Использовать только для обработки|
|S3|Низкая|Основное хранилище|
Использование HDFS значительно дороже.


## **6. Оптимизация затрат Spark-кластера**

### **Предложенные меры**
### **6.1. Уменьшение размера кластера**
#### **Было:**
```
data: 3 nodes (s3-c4-m16)
```

#### **Можно:**
```
data: 1-2 nodes
```

> Экономия: до **50–70%**

### **6.2. Использование более дешёвых дисков**

#### **Было:**
```
network-ssd
```

#### **Сделано:**
```
network-hdd
```

**Уже реализовано → экономия есть**

### **6.3. Отключение кластера вне работы**
Самый мощный способ оптимизации расходов:
- не держать кластер 24/7
- запускать только под задачу

> Экономия: до **90%**

---

### **6.4. Использование Object Storage вместо HDFS**
Уже частично реализовано через:
```
hadoop distcp s3a://...
```

Плюсы:
- дешевле
- не требует хранения на нодах

---

### **6.5. Уменьшение числа DataNodes**
Для учебного задания:
```
hosts_count = 1
```

> Полностью достаточно для демо

---

### **6.6. Spot/Preemptible VM (если бы было прод)**

> В Yandex Cloud:
- можно использовать прерываемые VM
- экономия до **70%**
